指南

创建上下文、配置源类型与异常处理。

配置：创建上下文

ParseContext 不提供源的读取方法，仅持有 getSource()、currentPosition() / setCurrentPosition(int) 以及可选的 currentScope。具体读取由解析器通过 ctx.getSource() 转为 ByteSource / CharSource 后调用其方法。

字节源

  • ParseContext.fromBytes(byte[])、fromBytes(byte[], offset, length)
  • fromByteBuffer(ByteBuffer)
  • 字符串按字节编码：fromEncodedBytes(text, Charset)、fromEncodedBytesUtf8(text)

字符源

  • 推荐：ParseContext.fromString(CharSequence)（不经编码，直接字符）
  • 或 fromChars(CharSequence)

对象源

  • ParseContext.fromObject(root)：自动设置以 root 为 target 的根 currentScope
  • 通过 (ObjectSource) ctx.getSource() 调用 getValue(expression) / getValue(expression, Class<R>)
  • 层级解析：enterScope(childTarget)、exitScope()

异常

ReadException

表示数据不足、越界等读取失败。可 addContextValue(label, value) 携带诊断信息，getMessage() 会包含上下文。组合子 optional、many、choose 在遇到 ReadException 时会回退位置并返回 null 或尝试下一项。

ParseException

表示格式或语义解析失败（例如 CSV 未闭合引号、业务校验失败）。同样支持 addContextValue。此类异常不会被 optional 等“吞掉”，会直接向外抛出。

类型与强转

解析器需根据当前上下文将 ctx.getSource() 转为 ByteSource 或 CharSource 再调用读方法。在非对应类型源上强转会抛出 ClassCastException。对象源上调用 setCurrentPosition 会抛出 UnsupportedOperationException。
